# Madrid Routing App

A client-server application for representing tree data and routing in Madrid, combining OSM data with local tree networks.

## Project Structure

- `backend/`: Python FastAPI server and data processing scripts.
  - `data/`: Raw and processed data (Excel, CSV, GraphML).
  - `scripts/`: Data processing, network generation, and routing scripts.
- `frontend/`: React app built with Vite, Leaflet, and Tailwind CSS.
- `web/`: Static HTML maps generated by Folium.

## Getting Started

### Prerequisites

- [uv](https://github.com/astral-sh/uv) (for Python dependency management)
- [Node.js](https://nodejs.org/) (for the frontend)

### 1. Start the Backend API

From the project root, run:

```bash
uv run uvicorn backend.main:app --reload
```

The API will be available at `http://localhost:8000`.

### 2. Start the Frontend

Navigate to the `frontend/` directory, install dependencies, and start the development server:

```bash
cd frontend
npm install
npm run dev
```

The app will be available at `http://localhost:5173`.

## Data Processing

To generate the networks used by the app, refer to the scripts in `backend/scripts/`.
